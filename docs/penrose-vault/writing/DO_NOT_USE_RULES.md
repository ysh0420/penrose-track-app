# Do Not Use Rules

These rules apply to Penrose writing, reports, dashboards, handoffs, and AI prompts.

## Do Not Use As Primary Source

- Conversation memory.
- v1 model portfolio, except as legacy archive.
- Stale page copy.
- One-name fixtures as system-wide evidence.
- Notion/Obsidian notes without Supabase or GitHub confirmation.

## Do Not Publish Or Expose

- Internal recommendations.
- Portfolio actions.
- Raw paid-data-derived rows.
- Source Run IDs or internal workflow IDs.
- Secrets, tokens, privileged keys, or auth details.
- Static internal report content embedded in the repo.

## Do Not Infer

- Sector RS when missing.
- Peer basket RS when missing.
- Global lead-lag when incomplete.
- Future portfolio from current 25 inherited holdings.
- Project-level conclusions from Zeon alone.

## Do Not Execute

- Browser AI execution.
- Trades.
- Rebalance.
- Portfolio state changes.
- Report publication without explicit approval.
- Destructive migrations without explicit approval.

## Required Replacement Behavior

- Use Supabase for current data and outputs.
- Use GitHub for code state.
- Use Track App as read-only view layer.
- Use this vault as operating map.
- Keep missing data visible.
